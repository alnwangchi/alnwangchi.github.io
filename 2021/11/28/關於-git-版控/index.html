關於 git 版控

雜紀

cmd + shift + . 顯示隱藏資料檔

CMD 操作

cd_ => 進入資料夾
cd.. => 回上層資料夾
ls => 顯示當前位置所有內容
~ => 當前位置所在
cmd+k => 清空面板
mkdir 資料夾名 => 新增資料夾
touch 檔案名 => 新增檔案

理解

當檔案有差異時，就可以被 commit，較好的 commit 習慣是一個功能完整後即可 commit，若積累一堆功能後才 commit 容易有想回到上一個版本但卻又回復太多的情形。

branch 就是讓個人可以獨立的作業做某個項目，

指令

設定 email
git config –global user.email “ro81516@gmail.com“
設定 使用者名稱
git config –global user.name “Allen.w”
檢查當前設定
git config –list

git init 第一步將資料夾初始化
git add . 將檔案加入 stage (.代表全部加上去)
git commit -m 'msg' 說明
git status 檢查狀態
git log 檢視各個 commit 點 亂碼代號
git branch allenB 建立新的分支 allenB
git checkout 檔名 將檔案回復到上一個 commit 點
git checkout 亂碼前四碼 該 commit
git checkout --hard 全部回復到上一個 commit 點
git checkout allenB 切到 allenB 分支上面
git checkout -b allenC 建立新的分支 allenC 順便切過去
git merge 當前分支去合併某分支 看我要把誰 merge 進來
git rebase 把當前分支的 commit 點拉到某分支前

當 merge 遇到衝突時就需要手動合併，當前頁面會顯示相撞的 HEAD 以及 income 內容讓我去判斷要如何處理，當選擇完畢後，還需要再度 add commit 一次

當推送分支的時候會遇到的問題，因為很可能本地不只有一個數據庫，如果直接用 git push 的話他會不確定到底是要推上 github 還是其他地方(正式主機、測試主機)，所以需要指定數據庫跟分支，那為何一開始直接 push 就會成功，那是因為 master 已經有預設了

git remote 檢查本地數據庫
git push origin branch 推送的完整指令 [預設遠端主機名][分支名稱]
